# 🖥️ VPS Resources Checker (VPSRC)

> **Автор:** Kvink  
> **Версия:** 2.6 (Persistent Background Logging)  
> **Лицензия:** MIT  

Простой, красивый и мощный инструмент для мониторинга ресурсов VPS/VDS. Работает из коробки, автоматически устанавливает зависимости и **продолжает собирать логи в фоне**, даже если вы закрыли терминал.

---

## ✨ Что нового в v2.6?
- 🔄 **Постоянное фоновое логирование**: Скрипт запускает демоны `atop` и текстового логгера, которые работают независимо от основного интерфейса.
- 📦 **Автоустановка `atop`**: Скрипт сам определит ваш пакетный менеджер (apt, dnf, yum, apk, pacman) и установит нужные утилиты.
- 🧹 **Умная очистка**: Логи старше 7 дней автоматически сжимаются (`.gz`), старше 30 дней — удаляются.
- 📜 **Гибкий просмотр**: Добавлена возможность просмотра текстовых логов за любую конкретную дату (с поддержкой чтения архивов на лету).
- 🛑 **Контроль демонов**: Новая команда в меню для полной остановки фоновых процессов.

---

## 🚀 Возможности
* 📊 **Дашборд в реальном времени**: CPU, RAM, Swap и Disk с цветовыми индикаторами и прогресс-барами.
* 📋 **Топ-5 процессов**: Мгновенное определение того, кто потребляет ресурсы.
* 🛡️ **Специализированный мониторинг**: Детальная статистика по процессам `olcRTC` и `x-ui`.
* ⚔️ **Детекция атак**: Анализ `journalctl` на наличие попыток атак (TLS handshake errors) на x-ui за последние 2 дня.
* 🖥️ **Диагностика GPU**: Базовая информация о графических ускорителях (drm/virtio_gpu).
* 📜 **История нагрузок**: Интерактивный просмотр бинарных логов `atop` за сегодня.

## ⚡ Установка и запуск

Скрипт устанавливается в систему одной командой и становится доступен глобально как `vpsrc`.

```bash
curl -s https://raw.githubusercontent.com/Kvinkgithub/VPSRC/master/vpsrc.sh -o /usr/local/bin/vpsrc
chmod +x /usr/local/bin/vpsrc
```

Запуск (требуются права root или sudo для установки atop и записи логов)
```bash
sudo vpsrc
```

📋 Интерактивное меню

После запуска вы увидите удобное меню с следующими опциями:

1
🔄 Обновить данные на экране

2
📊 olcRTC: Детали процессов (PID, CPU%, MEM%, RSS)

3
📊 x-ui: Детали процессов + анализ логов на атаки

4
📋 История (atop): Интерактивный просмотр лога за сегодня (управление: t/T — время, q — выход)

5
🧹 Очистка: Принудительное удаление логов старше 30 дней

6
📜 Текстовые логи: Просмотр отчета за конкретную дату (ГГГГ-ММ-ДД)

7
🛑 Стоп: Полная остановка фоновых процессов логирования

0
👋 Выход: Закрытие интерфейса (логи продолжат писаться в фоне!)


📂 Работа с логами

Все данные сохраняются в директорию:

📁 /var/log/cpu_monitor/
atop_YYYYMMDD.log / .gz — бинарные логи для глубокого анализа через atop -r.
text_YYYY-MM-DD.log / .gz — удобочитаемые текстовые сводки (Uptime, RAM, Disk, Top-5 CPU).

💡 Автоматическая очистка: При каждом запуске скрипт автоматически сжимает логи старше 7 дней и удаляет файлы старше 30 дней, экономя место на диске.
